# Шаблон INIT-Запиту

Використовуй цей шаблон лише тоді, коли проєкт ще не ініціалізовано.

Запит на ініціалізацію має починатися з `INIT:` і максимально прямо відповідати на запитання нижче.

## Мінімальний формат

```text
INIT: <коротка назва проєкту>

1. Що саме ми будуємо?
2. Чому це важливо?
3. Який перший конкретний deliverable?
4. Що зараз точно поза scope?
5. Який стек / платформа бажані?
6. Які середовища вже існують (local / test / stage / prod)?
7. Які зовнішні системи або API залучені?
8. Які модулі ти вже очікуєш?
9. Які критерії успіху?
10. Які є обмеження, дедлайни, бюджети або правила апруву?
11. Які важливі файли вже додані в `additionalContextFiles/` і що саме вони пояснюють?
```

## Що агент повинен зробити після валідного INIT

Після отримання валідного `INIT:` запиту агент має:

1. синтезувати відповідь у [`PROJECT_BRIEF.md`](PROJECT_BRIEF.md);
2. оновити [`PROJECT_STATUS.md`](PROJECT_STATUS.md) початковим відомим станом;
3. переглянути `additionalContextFiles/`, якщо там є матеріали, і відобразити релевантне в docs;
4. створити першу мапу модулів у [`MODULES.md`](MODULES.md);
5. декомпозувати роботу в дрібні задачі беклогу в [`BACKLOG.md`](BACKLOG.md);
6. залогувати факт ініціалізації в [`SESSION_LOG.md`](SESSION_LOG.md);
7. зафіксувати довговічні архітектурні вибори в [`DECISIONS.md`](DECISIONS.md).

## Важливе правило

Якщо беклог уже містить реальні задачі, а project brief вже ініціалізований, повторний `INIT:` не потрібен навіть на новій машині чи для нового ШІ агента.
